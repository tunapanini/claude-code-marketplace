# Commit and PR

현재 변경사항을 커밋하고, 테스트 후 PR을 생성하는 자동화 워크플로우입니다.

## 실행 단계

### 1. 커밋
- `git status`로 변경사항 확인
- `git diff`로 staged/unstaged 변경사항 확인
- 적절한 커밋 메시지 작성 후 커밋

### 2. 로컬 테스트
변경사항의 종류에 따라 적절한 테스트 실행:
- **테스트 코드 수정**: 해당 테스트 실행
- **기능/로직 수정**: 개발 서버 실행 후 관련 API/기능 요청하여 동작 확인
- **빌드/린트/설정 등 chore 작업**: 빌드 실행하여 성공 여부 확인
- 테스트 실패 시 중단하고 사용자에게 알림

### 3. 푸시
- 현재 브랜치를 원격에 푸시
- 필요 시 `-u` 플래그로 upstream 설정

### 4. PR 생성
- `gh pr create --draft` 사용 (draft PR로 생성)
- **제목**: 작업 유형 prefix 포함
  - `feat:` 새 기능
  - `fix:` 버그 수정
  - `chore:` 설정/빌드/의존성 등
  - `refactor:` 리팩토링
  - `docs:` 문서
  - `test:` 테스트
- **본문 구성**:
  - 변경사항 요약 (필수)
  - 해결하는 문제 (optional)
  - 테스트 계획 (optional)
  - 참고 - 스크린샷 등 (optional)

### 5. CI 체크
- `gh pr checks` 또는 `gh run list`로 CI 상태 확인
- CI 실패 시 로그 확인하고 사용자에게 알림

## 주의사항
- 각 단계 실패 시 즉시 중단
- 민감한 파일(.env 등) 커밋 방지
- force push는 절대 사용하지 않음
